# MemoArk v0.1.0 发布说明

> 状态：已完成。全量回归、真实浏览器验收、正式本地部署和 GitHub milestone 收口均已通过；本文件所在提交作为本轮唯一一次集中推送的交付批次。

本版优先解决三件事：升级前有可恢复备份、空账户可以迁入备忘录、网络中断时正文草稿尽量不丢失。

## 本版内容

### #1 SQLite 升级前备份

- 仅当**已有 SQLite 数据库确实存在待执行迁移**时，MemoArk 才会先创建备份；全新安装或无需迁移时不会重复备份。
- 备份完成后会执行 `PRAGMA integrity_check`。备份创建或校验失败时，迁移会在修改源数据库前停止。
- 默认 Docker 备份目录为 `/var/opt/memos/backups`，文件名包含迁移前后版本和 UTC 时间。
- 恢复前必须停止 MemoArk、校验备份并保留当前数据库副本。完整命令见：[SQLite 迁移备份与恢复操作说明](../operations/sqlite-migration-backups.md)。
- 当前只覆盖 SQLite；附件目录、MySQL 和 PostgreSQL 不在自动备份范围内。

### #2 从 MemoArk v1 JSON 导入

入口：**设置 → 账户 → 导入备忘录**。

- 只接受 `format = memoark.memo-export`、`formatVersion = 1` 的 JSON，文件最大 **64 MiB**。
- 只允许导入到没有任何备忘录的空账户；选择文件后先预检，再由用户确认导入。
- 支持恢复：正文 `content`、可见性 `visibility`、置顶 `pinned`、创建时间、更新时间、状态（正常/归档）以及从正文重新解析出的标签。
- 原备忘录 UID 不会沿用；导入时会生成新的 UID，避免与目标实例中的资源冲突。
- 不支持恢复：附件文件及附件记录、评论、关系、反应、位置和设置；预检会列出跳过数量。
- 导入采用单个数据库事务：格式错误、目标账户非空或写入中途失败时，不会留下半套导入数据。

### #3 正文草稿断网保留与重试

- 新建或编辑备忘录时，如果发生断网、服务不可用或请求超时，并且浏览器本地存储可用，正文会保存在当前设备的当前浏览器中，并显示“已保存在本机，尚未发送”。
- 刷新后可以恢复正文并手动重试；新建重试会复用同一请求 ID，降低重复创建的风险。只有服务端确认成功后才清除本地草稿。
- 本版只保证**正文文本**的本地恢复，不保证附件文件、仅附件草稿、完整离线浏览、后台自动同步或跨设备同步。
- 隐私模式、本地存储被禁用或空间不足时，草稿可能无法写入；页面会提示保持页面打开后重试。
- 权限、参数校验等服务端拒绝不会被误报为“离线已保存”，用户需按实际错误修正后再保存。

### #4 多标签 AND 筛选

- 可同时选择多个标签。
- 筛选关系为 **AND**：例如同时选择 `#工作` 和 `#重要`，只显示同时含有这两个标签的备忘录。

## 回滚验证清单

以下是故障时使用的操作手册。本次发布验证成功，未实际执行回滚，因此保持未勾选，不能把“已准备回滚资产”误写成“已做回滚”。

- [ ] 停止 MemoArk，确认没有进程继续写数据库。
- [ ] 将当前 `memos_prod.db`、`memos_prod.db-wal`、`memos_prod.db-shm`（如存在）复制到独立恢复目录。
- [ ] 对选定的迁移前备份执行 `PRAGMA integrity_check`，只在结果为 `ok` 时继续。
- [ ] 在应用已停止的前提下移除旧的 WAL/SHM 文件，并把已校验备份复制回 `memos_prod.db`。
- [ ] 恢复文件属主/权限，启动与该备份源版本兼容的上一版镜像；不能只降级镜像而继续使用已升级数据库。
- [ ] 验证登录、近期备忘录读取、创建/编辑和日志，再决定是否删除回滚前保留的数据库副本。
- [ ] 回滚失败时保持服务停止，不反复覆盖数据库，使用独立恢复副本继续排查。

详细恢复步骤以 [SQLite 迁移备份与恢复操作说明](../operations/sqlite-migration-backups.md) 为准。
