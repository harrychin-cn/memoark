# MemoArk 功能使用说明

**中文说明** | [English Guide](user-guide-en.md)

本文面向直接安装 Windows 或 Android 版本的普通用户。两种版本都把后端、前端和 SQLite 数据库放在本机，不需要
Docker、Node.js、Go、VPS 或外部服务器。

## 1. 下载与安装

从 [GitHub 最新版本页面](https://github.com/harrychin-cn/memoark/releases/latest) 下载适合自己的文件：

- Windows 普通用户：`MemoArk-Setup.exe`
- Windows 便携使用：文件名以 `windows-amd64.zip` 结尾的压缩包
- Android 手机：文件名以 `Android.apk` 结尾的安装包
- 校验下载：`SHA256SUMS.txt`

### Windows 安装版

1. 双击 `MemoArk-Setup.exe`，按向导完成安装。
2. 安装程序会创建桌面和开始菜单快捷方式。
3. 双击 **MemoArk**。程序在后台启动，不会一直显示命令行窗口。
4. 浏览器会打开本机页面；第一次使用时创建管理员账号。

默认数据目录为 `%LOCALAPPDATA%\MemoArk`，里面包含 SQLite 数据库、附件和日志。升级或卸载程序不会删除这个目录。

需要完全停止程序时，使用开始菜单中的 **退出 MemoArk**，不要直接强制结束正在写入数据的进程。

### Windows 便携版

1. 将 ZIP 完整解压到普通文件夹，不要直接在压缩包内运行。
2. 双击 `START-MemoArk.cmd`。
3. 保持命令窗口打开；浏览器会自动打开 MemoArk。
4. 使用完毕后，在命令窗口按 `Ctrl+C`，等待程序退出。

便携版默认也使用 `%LOCALAPPDATA%\MemoArk` 保存数据，因此可以和安装版读取同一份本机数据，但不要同时启动两个实例。

### Android 安装版

1. 下载 APK 后，按系统提示允许浏览器或文件管理器“安装未知应用”。
2. 安装并打开 MemoArk；第一次使用时创建管理员账号。
3. 后续升级直接安装同一发布来源的新 APK，不要先卸载旧版本。

Android 数据保存在应用私有目录。覆盖升级会保留数据，但卸载 App 通常会同时删除数据，卸载前必须先导出需要保留的备忘录。

## 2. 首次设置

### 创建账号

首次打开时显示“创建您的账户”。填写用户名和密码后注册，首个账号是本机实例管理员。请妥善保管密码。

### 切换语言和主题

- 未登录时：使用登录或注册页面底部的语言和主题菜单。
- 已登录后：进入 **设置 → 偏好设置 → 外观**。
- 主题支持：跟随系统、浅色、深色、纸张。
- 语言和主题修改后会立即生效；登录状态下的语言选择会保存到账号。

## 3. 创建和编辑备忘录

1. 回到 **主页**，点击编辑框并输入内容。
2. 选择可见性：
   - **私有**：只有自己可以查看。
   - **工作区**：登录到同一实例的成员可以查看。
   - **公开**：无需登录也能查看；管理员可以禁用公开备忘录。
3. 点击 **保存**。

编辑器支持 Markdown，例如：

```markdown
# 标题
- 普通列表
- [ ] 待办事项
**粗体**、`代码` 和 https://example.com
```

输入 `#标签名` 可以添加标签。输入 `/` 可以打开命令提示。插入菜单可用于：

- 上传图片、音频、视频或文档附件
- 添加位置
- 链接另一条备忘录
- 录制音频附件

录音需要授予麦克风权限；“转为文字”还需要管理员先配置可用的 AI provider。

### 草稿保护

编辑中的未保存正文会缓存在当前设备上：

- 意外刷新或退出后，再次编辑时可以选择 **恢复** 或 **丢弃**。
- 如果服务器版本已经变化，会显示冲突提示；恢复后先核对内容，再手动保存。
- 发送失败时可以保留本机草稿并重试。

草稿缓存只属于当前设备和浏览器，不等同于完整数据备份。

## 4. 查找和整理

- **搜索**：按正文内容查找备忘录。
- **过滤器**：按标签、可见性、创建者、是否包含代码、链接或待办等条件筛选。
- **排序**：按创建时间或更新时间，以最新或最早优先显示。
- **日历**：按日期查看当天的备忘录。
- **置顶**：把常用内容固定在前面。
- **归档**：从主页移走但继续保留，可在 **已归档** 页面恢复。
- **删除**：永久删除备忘录以及相关附件、链接和引用，无法撤销。
- **捷径**：保存常用过滤条件，之后一键重新使用。

## 5. 附件、评论和通知

### 附件

编辑备忘录时选择 **上传文件**。Android 会打开系统文件选择器，Windows 会打开浏览器文件选择窗口。

在 **附件** 页面可以按媒体、音频和文档分类查看文件，也可以清理没有关联到任何备忘录的上传文件。

### 评论、提及和通知

管理员启用评论后，可以在备忘录详情页发表评论。正文或评论中的成员提及会出现在 **通知** 页面；通知可以归档。

## 6. 分享

打开备忘录的更多操作菜单后，可以：

- 复制正文或普通链接
- 创建带有效期的分享链接
- 查看和撤销已经创建的分享链接
- 将备忘录生成 PNG 图片
- 在 Android 上调用系统分享菜单分享文字或图片

私有或工作区备忘录不会因为普通页面链接而自动公开；需要创建公开访问入口时使用专门的分享链接，分享完成后可以随时撤销。

Windows 和 Android 本地版只监听本机地址，因此复制出来的页面链接或分享链接通常只能在当前设备打开，不能让其他人的设备访问。
本地版需要对外发送内容时，优先使用分享图片、复制正文或 Android 系统分享。只有部署在对方可以访问的服务器上时，分享链接才适合直接发给别人。

## 7. 导入、导出和备份

进入 **设置 → 我的账号**：

- **导出备忘录**：下载当前账号全部普通和归档备忘录的 JSON 文件。
- **导入 MemoArk JSON**：先预检，再导入到没有任何备忘录的空账号。

当前 JSON 导出包含备忘录数据和附件元数据，但不包含附件文件、评论和实例配置。导入也不会恢复附件、评论、关系、反应、位置和设置；
因此 JSON 适合迁移文字备忘录，不是完整磁盘备份。导入文件最大为 64 MiB。

### Windows 完整备份

1. 使用开始菜单中的 **退出 MemoArk**，确认程序已经停止。
2. 完整复制 `%LOCALAPPDATA%\MemoArk` 到其他磁盘。
3. 同时保留最近一次 JSON 导出，便于单独恢复文字内容。

恢复完整备份前先退出 MemoArk，再还原整个数据目录。不要在程序运行时替换 SQLite 数据库。

### Android 备份

Android 普通用户无法直接复制 App 私有数据库。请定期使用 **导出备忘录**，通过系统保存窗口把 JSON 放到手机文档目录、云盘或其他设备。
由于 JSON 不包含附件文件，重要附件还应单独保留原文件。卸载 App 前必须完成导出。

## 8. 设置说明

普通账号可以使用：

- 我的账号：头像、昵称、密码、导入导出、访问令牌
- 偏好设置：语言、主题、默认可见性、显示时间
- Webhook 和已关联身份

管理员还可以管理：

- 成员和注册规则
- 实例名称、公开访问和系统设置
- 备忘录评论、位置、敏感内容和表态
- 标签、附件存储、邮件通知、SSO、AI provider 和资源统计

更改存储、SSO、AI 或访问令牌等高级配置前，先记录原值并做好数据备份。

## 9. 手机操作说明

- **返回键**：优先关闭当前弹窗或返回上一页；没有可返回页面时退出 App。
- **选择文件**：上传附件或导入 JSON 时使用 Android 系统文件选择器。
- **保存文件**：导出 JSON 或图片时使用系统保存窗口选择位置。
- **系统分享**：支持将文字或生成的图片交给微信、邮件、网盘等已安装应用。
- **切换后台或锁屏**：App 会停止本地后端并安全关闭数据库；回到 App 时自动重新启动。

## 10. 升级和卸载

### Windows

- 升级前退出 MemoArk，直接运行新版 `MemoArk-Setup.exe` 覆盖安装。
- 卸载可以使用 Windows“已安装的应用”或开始菜单中的 **卸载 MemoArk**。
- 卸载只删除程序、快捷方式和注册信息，不删除 `%LOCALAPPDATA%\MemoArk` 用户数据。
- 确定不再需要数据时，再手动删除数据目录。

### Android

- 直接安装新版 APK 覆盖升级，签名一致时账号、数据库和附件会保留。
- 不要通过“先卸载再安装”的方式升级。
- Android 卸载通常会删除账号、数据库和附件，卸载前先导出。

## 11. 常见问题

### Windows 启动后没有打开页面

从开始菜单重新打开 MemoArk，或在浏览器访问 `http://127.0.0.1:5230/`。如果仍无响应，先运行 **退出 MemoArk**，再重新启动。

### 页面提示端口被占用

通常是已有一份 MemoArk 正在运行。不要让两个程序同时使用同一数据目录；先退出旧实例后重试。

### Android 上传或录音失败

检查系统是否允许 MemoArk 访问所选文件或麦克风，然后重新选择文件或开始录音。

### 切换语言后仍有旧文字

确认已经安装最新版本，在 **设置 → 偏好设置** 中重新选择语言，然后完全退出并重新打开 App。若仍能稳定复现，请提交问题并附上页面截图、
MemoArk 版本、系统版本和已选语言。
